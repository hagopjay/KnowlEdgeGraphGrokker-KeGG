Cron / Time-Based Automation SCHEDULING
Cron Expression Cheat Sheet
FORMAT:  min  hour  dom  month  dow
         0-59  0-23  1-31  1-12  0-7 (0,7=Sun)

# ── Daily ──────────────────────────────
0 7 * * *        → Daily 7:00 AM UTC
0 7 * * 1-5      → Weekdays 7:00 AM UTC
30 8 * * 1-5     → Weekdays 8:30 AM UTC
0 9,17 * * 1-5   → Weekdays 9am and 5pm

# ── Interval ───────────────────────────
*/15 * * * *     → Every 15 minutes
*/30 * * * *     → Every 30 minutes
0 */2 * * *      → Every 2 hours
0 */1 * * *      → Every hour

# ── Weekly ─────────────────────────────
0 9 * * 1        → Monday 9am UTC
0 20 * * 0       → Sunday 8pm UTC
0 1 * * 0        → Sunday 1am UTC

# ── Monthly ────────────────────────────
0 0 1 * *        → 1st of month midnight
0 9 1 * *        → 1st of month 9am
0 10 1 * *       → Monthly audit cron

# ── Timezone correction (CST = UTC-6) ──
0 13 * * 1-5     → 7am CST (CST = UTC-6)
0 12 * * 1-5     → 7am CDT (CDT = UTC-5)

# ── Common use cases ───────────────────
0 7 * * 1-5      morning briefing (weekday)
0 */2 * * *      heartbeat check
30 17 * * 1-5    EOD summary
0 18 * * 1-5     meeting notes processor
0 21 * * *       screenshot cleanup
30 16 * * 5      Friday weekly report
0 8 * * 1        Monday content/task review
Timezone gotcha: Cron runs in the server timezone. Use "timezone": "America/Chicago" in config, or manually convert to UTC. Always annotate which timezone in task description.

✅ GOOD — specific paths, clear outputs, explicit state tracking

HEARTBEAT.md checklist
# Heartbeat Protocol
## Checklist (runs every 2h)
1. Check active project sessions for pending results
2. Review /clawd/logs/errors.log for unaddressed errors
3. Disk usage — alert if above 80%
4. Verify Gateway healthy
5. Check unanswered operator messages
6. Update /clawd/memory/heartbeat-YYYY-MM-DD.log

## Alert Conditions
- Any unaddressed ERROR → WhatsApp alert
- Disk >80% → WhatsApp warning
- Project stuck >24h in same state → WhatsApp alert

## Normal Operation
No outbound messages if all clear. Just log.
Heartbeat interval trade-off: Every 15min = 8× cost of 2hr for same coverage. Use 2hr unless you need <2hr incident response.

Failure Modes + Mitigations
🔴 Page structure changed
Re-snapshot. Target by role/text not position. "Click the button that says 'Add to Cart'" not "third button in .product-actions div"
🔴 Rate limiting (429 / CAPTCHA)
Add delays between requests. setTimeout(2000) between fetches. Don't hammer. Respect Retry-After header.
🔴 Login session expired
Detect "Please log in". Re-auth from /clawd/secrets/credentials.json. Store cookies. Never hardcode creds — chmod 600 on secrets.
🔴 JS-heavy + anti-bot protection
1. Check for official API. 2. Use chrome profile with realistic delays. 3. Find alternative source. 4. Accept: this automation isn't feasible.
Login flow template
1. navigate {login_url}
2. snapshot (get refs)
3. fill email ref ← /clawd/secrets/credentials.json
4. fill password ref ← credentials.json
5. click submit ref
6. wait: "Loading" disappear OR "Dashboard" appear
7. snapshot → confirm login

AGENTS.md Full Schema
# AGENTS.md
## Role
[Main agent's single responsibility — routing only]

## Named Agents
### research-lead
**Role:** Research orchestrator
**Triggers:** Tasks requiring external research/data
**Capability:** Spawns N parallel research sub-agents
**Output:** /clawd/research/{project-id}/
**Reports to:** Main router

### project-lead
**Role:** Implementation orchestrator
**Triggers:** Research-complete tasks ready for building
**Capability:** Sprint plans + impl sub-agents
**Output:** /clawd/projects/{project-id}/
**Reports to:** Main router

### test-lead
**Role:** QA orchestrator (TEA: Test-Execute-Assess)
**Triggers:** Implementations awaiting validation
**Capability:** Runs tests, generates audit report
**Output:** /clawd/projects/{project-id}/test-artifacts/
**Reports to:** Main router

## Routing Rules
- "write" | "draft" → writer agent
- "research" | "find" | "check" → researcher
- "code" | "build" | "fix" → coder

## Quality Gates
Before routing to next phase, verify:
□ Required artifact files exist at expected paths
□ Files are non-empty
□ Gate decision in summary.md = PASS|READY

## RALPH Escalation Protocol
On BLOCKED/FAILED from sub-agent:
1. Log → /clawd/logs/escalations.log:
   TIMESTAMP | Project {id} | Phase {phase} | Error | Retry {n}
2. Retry (if n < 3): spawn new agent,
   "Previous attempt failed with {error}. Different approach."
3. Escalate (n >= 3 OR unrecoverable):
   WhatsApp: 🚨 BLOCKED: {id} | {phase} | {error} | Awaiting decision
4. HOLD: no further progress until operator responds

## Memory Protocol
After each phase completion, write to:
/clawd/memory/YYYY-MM-DD.md (timestamped action log)
Update /clawd/status-board.md with project status
Subagent Spawning Patterns
# Spawn a single worker
Spawn a sub-agent labeled "research-phase-1" to:
  [fully specified task with exact paths]
  Report back when complete.

# Spawn N parallel workers simultaneously
Spawn three parallel research agents:
1. Label: research-competitor-a
   Task: Research Competitor A pricing and features.
   Save to /clawd/research/20260222/competitor-a.md
2. Label: research-competitor-b
   Task: Research Competitor B pricing and features.
   ...
Wait for all three to complete.
Compile findings into comparison report.

# Steer a running agent
{
  "tool": "subagents",
  "action": "steer",
  "target": "research-task-1",
  "message": "Also check Product Hunt for recent launches."
}

# Kill a stuck agent
{ "tool": "subagents", "action": "kill",
  "target": "research-task-1" }

# List running agents
{ "tool": "subagents", "action": "list",
  "recentMinutes": 30 }
Parallel speedup example: 3 sequential research tasks × 5min = 15min. 3 parallel workers = 5min total (3× speedup).
Agent Failure Patterns + Fixes
Failure TypeSignsFix
Context overflow Repetitive responses, "forgets" earlier instructions, incoherent output Split task into smaller chunks. "Research aspect 1" + "Research aspect 2" as separate agents.
Going off-script Unexpected output files, actions outside stated scope Be prescriptive: "Your ONLY job is X. Do NOT write code. Do NOT do anything else. Save ONLY to {path}."
Lost results Agent reports "complete" but artifact files missing Always validate: ls -la /clawd/projects/{id}/research-artifacts/. Never trust verbal "done".
Infinite tool loops Excessive repeated tool calls, task never completing Add stopping conditions: "Research until 3+ credible sources OR 30 minutes elapsed. Then compile and report. Do not continue indefinitely."
Handoff failure Receiving agent starts on incomplete/missing artifacts Use structured handoff: explicit artifact paths, gate check before routing, handoff message format with project-id + expected outputs.
Structured handoff message format
Project {id}: {Phase} phase
Research gate: READY
Research summary: /clawd/projects/{id}/research-artifacts/research-summary.md
Your task: Execute planning phase
Output dir: /clawd/projects/{id}/planning-artifacts/
Expected artifacts: prd.md, architecture.md, planning-summary.md
When complete: set gate in planning-summary.md + report back.

# Pattern 1: Retry with exponential backoff
def with_retry(fn, max_attempts=3, delay=2):
    for attempt in range(1, max_attempts + 1):
        try:
            return fn()
        except Exception as e:
            if attempt == max_attempts: raise
            wait = delay ** attempt
            logging.warning(
              f"Attempt {attempt} failed: {e}. Retry {wait}s")
            time.sleep(wait)

# Pattern 2: Graceful degradation
def get_with_fallback(primary_path, fallback_path):
    try:
        return fetch_fresh_data(primary_path)

            return json.loads(Path(fallback_path).read_text())

Web Scraping at Scale
import time, requests, hashlib, json
from pathlib import Path
from urllib import robotparser
from urllib.parse import urlparse

def check_robots(url):
    parsed = urlparse(url)
    rp = robotparser.RobotFileParser()
    rp.set_url(f"{parsed.scheme}://{parsed.netloc}/robots.txt")
    try: rp.read(); return rp.can_fetch("*", url)
    except: return True  # assume allowed if check fails

def polite_get(url, delay=2):
    if not check_robots(url):
        raise ValueError(f"robots.txt blocks: {url}")
    r = requests.get(url, headers={
        "User-Agent": "Mozilla/5.0 (compatible; bot/1.0)"
    }, timeout=30)
    time.sleep(delay)  # politeness delay
    return r

# URL-based pagination
def scrape_all_pages(base_url, max_pages=50):
    results = []
    for page in range(1, max_pages + 1):
        data = parse_page(polite_get(
            f"{base_url}?page={page}").text)
        if not data: break
        results.extend(data)
    return results

# Response caching (TTL-based)
CACHE_DIR = Path("/clawd/data/web-cache")
def get_cached_or_fetch(url, ttl_hours=24):
    key = hashlib.md5(url.encode()).hexdigest()
    f = CACHE_DIR / f"{key}.json"
    if f.exists():
        cached = json.loads(f.read_text())
        if (time.time()-cached["timestamp"])/3600 < ttl_hours:
            return cached["content"]
    content = polite_get(url).text
    CACHE_DIR.mkdir(parents=True, exist_ok=True)
    f.write_text(json.dumps({
        "url": url, "timestamp": time.time(),
        "content": content
    }))
    return content

Disk Guardian Pattern
#!/usr/bin/env python3
"""disk-guardian.py — Monitor disk thresholds"""
import shutil, json
from datetime import datetime
from pathlib import Path

WATCH_PATHS = [
    {"path": "/", "warn": 80, "crit": 90},
    {"path": Path.home() / "clawd", "warn": 85, "crit": 95}
]

def check_path(cfg):
    usage = shutil.disk_usage(cfg["path"])
    pct = (usage.used / usage.total) * 100
    status = (
        "CRITICAL" if pct >= cfg["crit"] else
        "WARNING"  if pct >= cfg["warn"] else
        "OK"
    )
    return {"path": str(cfg["path"]),
            "percent": round(pct, 1),
            "free_gb": round(usage.free/1e9, 1),
            "status": status}

results = [check_path(p) for p in WATCH_PATHS]
print(json.dumps({
    "timestamp": datetime.now().isoformat(),
    "results": results,
    "has_critical": any(r["status"]=="CRITICAL" for r in results),
    "has_warning":  any(r["status"]=="WARNING"  for r in results)
}, indent=2))
# Cron: every 3 hours
{
  "schedule": "0 */3 * * *",
  "task": "Run disk-guardian.py. If has_critical:
  immediate WhatsApp. If has_warning and 8-20h:
  WhatsApp warning. Otherwise log only."
}
